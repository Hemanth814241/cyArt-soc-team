Threat Hunting Data Sources – Week 4

Targeted MITRE ATT&CK Technique:
T1078 – Valid Accounts

Primary Data Sources:
1. Windows Security Event Logs
   - Event ID 4672 (Special Privileges Assigned to New Logon)
   - Event ID 4624 (Successful Logon)
   - Purpose: Identify abnormal privilege assignments and suspicious login activity.

2. Wazuh SIEM Logs
   - Authentication events
   - Privilege escalation alerts
   - Purpose: Correlate host-level security events across endpoints.

3. Elastic Security Logs
   - Centralized log aggregation
   - Timeline analysis of authentication and authorization events
   - Purpose: Detect anomalies across multiple systems.

4. Threat Intelligence Feeds
   - AlienVault OTX
   - Purpose: Validate suspicious IP addresses or indicators associated with
     credential abuse or unauthorized access.

Expected Indicators:
- Unexpected admin privilege assignment
- Privileged logons outside business hours
- Privilege escalation without a corresponding change request

Outcome:
These data sources enable proactive detection of valid account abuse by
correlating authentication, privilege, and threat intelligence data.
